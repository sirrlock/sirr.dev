# API Reference

Sirr exposes HTTP endpoints for secrets management, audit logging, webhooks, and scoped API keys. All endpoints except `/health` require a Bearer token — either your `SIRR_MASTER_KEY` or a [scoped API key](/api-keys). Requests and responses use JSON. {{ className: 'lead' }}

<Note>
  All authenticated endpoints require the `Authorization: Bearer <SIRR_MASTER_KEY>` header. The master key is set when you start the server and is used both for authentication and as the seed for encryption.
</Note>

---

## Create a public secret {{ tag: 'POST', label: '/secrets', id: 'post-secrets' }}

<Row>
  <Col>

    Create an anonymous (public) encrypted secret. The server generates a 256-bit hex ID and returns it along with a shareable URL. No `key` field — this is a dead drop.

    ### Required attributes

    <Properties>
      <Property name="value" type="string">
        The secret value to encrypt and store.
      </Property>
    </Properties>

    ### Optional attributes

    <Properties>
      <Property name="ttl_seconds" type="integer">
        Time-to-live in seconds. The secret is automatically destroyed after this duration. Omit for no expiration.
      </Property>
      <Property name="max_reads" type="integer">
        Maximum number of times the secret can be read. The secret is permanently destroyed after this many reads. Omit for unlimited reads.
      </Property>
    </Properties>

    <Note>
      Without a valid license key, Sirr allows a maximum of 100 active secrets per instance. Attempting to exceed this limit returns a `402 Payment Required` error.
    </Note>

  </Col>
  <Col sticky>

    <CodeGroup title="Request" tag="POST" label="/secrets">

    ```bash {{ title: 'cURL' }}
    curl -X POST http://localhost:39999/secrets \
      -H "Authorization: Bearer $SIRR_MASTER_KEY" \
      -H "Content-Type: application/json" \
      -d '{
        "value": "postgres://user:pass@db:5432/myapp",
        "ttl_seconds": 3600,
        "max_reads": 1
      }'
    ```

    ```js {{ title: 'Node.js' }}
    const response = await fetch('http://localhost:39999/secrets', {
      method: 'POST',
      headers: {
        'Authorization': `Bearer ${process.env.SIRR_MASTER_KEY}`,
        'Content-Type': 'application/json',
      },
      body: JSON.stringify({
        value: 'postgres://user:pass@db:5432/myapp',
        ttl_seconds: 3600,
        max_reads: 1,
      }),
    })

    const data = await response.json()
    ```

    ```python {{ title: 'Python' }}
    import requests

    response = requests.post(
        "http://localhost:39999/secrets",
        headers={"Authorization": f"Bearer {SIRR_MASTER_KEY}"},
        json={
            "value": "postgres://user:pass@db:5432/myapp",
            "ttl_seconds": 3600,
            "max_reads": 1,
        },
    )

    data = response.json()
    ```

    </CodeGroup>

    ```json {{ title: 'Response (201)' }}
    {
      "id": "a3f87b2c9d4e5f6a1b2c3d4e5f6a7b8c9d0e1f2a3b4c5d6e7f8a9b0c1d2e3f4",
      "url": "https://sirrlock.com/s/a3f87b2c..."
    }
    ```

    ```json {{ title: 'Error (402)' }}
    {
      "error": "secret limit reached",
      "message": "Free tier allows up to 100 active secrets. Set SIRR_LICENSE_KEY to unlock unlimited usage."
    }
    ```

  </Col>
</Row>

## List secrets {{ tag: 'GET', label: '/secrets', id: 'get-secrets' }}

    const response = await fetch('http://localhost:39999/secrets', {
      headers: {
        'Authorization': `Bearer ${process.env.SIRR_MASTER_KEY}`,
      },
    })

    const data = await response.json()

    import requests

    response = requests.get(

## Org-Scoped API (Multi-Tenant) {{ id: 'org-scoped-api' }}

Sirr supports multi-tenant isolation through organizations, principals, and role-based access control. Org-scoped endpoints live under `/orgs/{org_id}/` and require authentication via either the master key (`SIRR_API_KEY`) or a principal key. Master key holders have full access; principal key holders are limited by their assigned role's permissions.

<Note>
  Organization management (creating/deleting orgs, principals, and roles) requires the master key. Day-to-day secret operations use principal keys with role-based permissions.
</Note>

## Create an org secret {{ tag: 'POST', label: '/orgs/:org_id/secrets', id: 'post-orgs-secrets' }}

<Row>
  <Col>

    Create a secret within an organization scope. Requires the `SecretCreate` permission (`c`).

    ### Path parameters

    <Properties>
      <Property name="org_id" type="string">
        The organization to store the secret in.
      </Property>
    </Properties>

    ### Required attributes

    <Properties>
      <Property name="key" type="string">
        Unique identifier for the secret within this org.
      </Property>
      <Property name="value" type="string">
        The secret value to encrypt and store.
      </Property>
    </Properties>

    ### Optional attributes

    <Properties>
      <Property name="ttl_seconds" type="integer">
        Time-to-live in seconds. The secret is automatically destroyed after this duration.
      </Property>
      <Property name="max_reads" type="integer">
        Maximum number of reads before the secret is burned.
      </Property>
      <Property name="delete" type="boolean">
        Whether to burn (`true`, default) or seal (`false`) on max_reads. Sealed secrets block reads but allow PATCH.
      </Property>
      <Property name="allowed_keys" type="array">
        List of principal key IDs allowed to read this secret. Omit for org-wide access.
      </Property>
    </Properties>

  </Col>
  <Col sticky>

    <CodeGroup title="Request" tag="POST" label="/orgs/:org_id/secrets">

    ```bash {{ title: 'cURL' }}
    curl -X POST http://localhost:39999/orgs/org_a1b2c3d4/secrets \
      -H "Authorization: Bearer $SIRR_PRINCIPAL_KEY" \
      -H "Content-Type: application/json" \
      -d '{
        "key": "DB_URL",
        "value": "postgres://user:pass@db:5432/myapp",
        "ttl_seconds": 3600,
        "max_reads": 5
      }'
    ```

    </CodeGroup>

    <CodeGroup title="Response">

    ```json {{ title: '201 Created' }}
    {
      "key": "DB_URL"
    }
    ```

    ```json {{ title: '409 Conflict' }}
    {
      "error": "secret_exists",
      "message": "Key 'DB_URL' already exists. Delete it first."
    }
    ```

    </CodeGroup>

  </Col>
</Row>
